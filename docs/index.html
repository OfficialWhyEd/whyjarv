capabilities

Ask anything. Do anything.

WhyJarv is not a chatbot. It is a voice-activated agent with direct access to your Mac. Say it — it happens.

Mac control
"Open Xcode and show me the last build errors"
AppleScript opens Xcode, Claude reads the build output, speaks the errors back to you.
Memory
"Remember that I hate meetings before 10am"
Stored in SQLite. TF-IDF retrieval surfaces it whenever you ask about your schedule.
Code
"Write a commit message for the changes in this repo"
Claude reads the diff via MCP, drafts a conventional commit, pastes it to the terminal.
Context
"What did we discuss yesterday about WhyPost?"
TF-IDF retrieval over conversation history returns the exact exchange in milliseconds.
System
"Play something calm on Spotify and dim the screen to 30%"
Claude fires two AppleScript calls simultaneously — Spotify play + display brightness.
Reasoning
"I have a call in 40 minutes and haven't read the brief. Give me the three things I need to know."
Claude reads the file, fetches calendar context via MCP, synthesizes a 3-point brief and speaks it. Groq handles the voice reply before Claude finishes reading — you hear the first sentence in 247ms.

philosophy

Built around one principle.

Speed is a feature. Locality is a feature. Memory is a feature. None of them should require a monthly subscription.

01

Race mode

Two AI models fire simultaneously the moment you finish speaking. LLaMA 70B on Groq and Gemini Flash race in parallel over separate HTTP connections. The first HTTP chunk back wins — the other connection is silently cancelled.


Nobody does this. The result is sub-300ms conversational latency on a 2015 MacBook Pro. When a real action is needed, Claude Code CLI takes over via MCP. You hear the answer first; the action follows.

02

Mac-native

Apple TTS and STT. AppleScript for system actions. Rumps menu bar icon. No Electron. No browser required after startup.

03

Local memory

SQLite + TF-IDF retrieval. Every conversation indexed. Preferences, context, history — available forever, offline.

04

Real actions

Claude CLI runs with full MCP access. Opens apps, writes and runs code, browses the web, reads your files.

05

Zero cloud storage

No conversation data leaves your Mac. No account required. Your API keys stay in .env locally.

technical

Everything under the hood.

FastAPI backend, Three.js frontend, native Apple voice. Every layer built to minimize latency.

voice

Apple native STT/TTS

Web Speech API for recognition, speechSynthesis for output. Works offline for TTS. Zero third-party voice service.

backend

FastAPI + WebSocket

Python on port :8340. Real-time bidirectional communication with the Three.js frontend via WebSocket.

frontend

Three.js orb

React frontend with a signal-orange particle orb that pulses to voice activity and visualizes the AI pipeline state.

memory

SQLite + TF-IDF

memory_store.py — atomic writes, TF-IDF indexed retrieval. Finds relevant context from months of conversation instantly.

mcp

Claude Code CLI

Real Mac actions: opens apps, writes code, browses web, reads files. Full MCP tool access. 3–8s for complex actions.

menu bar

macOS menu bar

Rumps-based native icon. Start and stop the assistant without a window. Launches silently, uses zero resources while idle.
